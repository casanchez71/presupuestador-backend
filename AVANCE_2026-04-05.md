## 3. QUÉ FALTA (LO CRÍTICO)

### El flujo automático de punta a punta NO EXISTE todavía

Hoy: IA analiza plano → genera items genéricos → NO tienen recursos → NO se calculan automáticamente → hay que asignar precios manualmente.

**Lo que se necesita (descubierto analizando los Excels):**

El Excel del arquitecto tiene 3 capas:

```
CAPA 1: CATÁLOGOS (00_Mat, 00_MO, 00_Eq, 00_Sub)
   ↓ VLOOKUP por código
CAPA 2: ANÁLISIS DE PRECIO UNITARIO (hojas X.Y)
   Cada item tiene: materiales + mano de obra + equipos + materiales indirectos + subcontratos
   Cada recurso: código → precio del catálogo × cantidad × desperdicio
   Total de la sección ÷ cantidad del item = PRECIO UNITARIO
   ↓ se lleva a la planilla
CAPA 3: PRESUPUESTO FINAL (01_C&P)
   Cantidad del plano × Precio unitario = Costo directo
   + Indirectos (14%) + Beneficio (25%) + Impuestos = NETO FINAL
```

### Los 5 gaps específicos:

1. **No existe "Análisis de Precio Unitario" en el sistema**
   - El Excel tiene una hoja por item con composición detallada (materiales, MO, equipos, etc.)
   - El sistema solo tiene `item_resources` plano, sin la estructura de 5 secciones
   - Falta: factor de desperdicio, cargas sociales, producción por día

2. **La IA no conoce los catálogos**
   - Los Excels usan VLOOKUP a las hojas 00_* con códigos específicos (H30, HADN6, MO-CA, etc.)
   - La IA genera nombres genéricos sin códigos → no hay match posible

3. **La IA no genera recursos/composición**
   - Solo genera items (nombre + cantidad + unidad)
   - Debería generar la composición: "para 1m3 de H21 necesitás X cemento, Y arena, Z hierro"

4. **No hay cálculo en cascada automático**
   - Recurso → precio unitario compuesto → × cantidad → directo → + indirectos → neto
   - Hoy hay que apretar "Recalcular" manualmente

5. **Indirectos no reflejan la estructura real**
   - El Excel usa: Imprevistos 3% + Estructura 5.6% + Jefatura 5.6% = ~14.2%
   - Luego Beneficio 25% SOBRE el subtotal con indirectos (no sobre directo)
   - Luego Ingresos Brutos 7% + Imp. Cheque 1.2% SOBRE el subtotal con beneficio
   - Luego IVA 21%
   - El sistema actual aplica beneficio sobre directo, no sobre subtotal con indirectos

## 4. PLAN DE IMPLEMENTACIÓN ACTUALIZADO

### Etapa 1: Modelo de datos — Análisis de Precio Unitario
Crear tabla `unit_price_analysis` que replica las hojas X.Y del Excel:
- Vinculada a cada budget_item
- 5 secciones: materiales, mo_personas, mo_equipos, mo_materiales_indirectos, subcontratos
- Cada fila: catalog_code, descripcion, unidad, cantidad, factor_desperdicio, cantidad_con_desperdicio, precio_unitario, subtotal
- Para MO: workers × days × (1 + cargas_sociales) × jornal
- Totales por sección ÷ cantidad del item = precio unitario por categoría

### Etapa 2: IA con contexto de catálogo
- Al analizar un plano, incluir en el prompt los códigos y descripciones del catálogo del usuario
- La IA genera items CON composición de recursos que matchean códigos del catálogo
- Output esperado:
```json
{
  "items": [{
    "code": "1.1", "description": "Hormigón H30 columnas",
    "unit": "m3", "quantity": 23,
    "resources": {
      "materiales": [
        {"catalog_code": "H30", "qty_formula": "=QTY*1.10", "waste": 0.10},
        {"catalog_code": "HADN6", "qty": 227, "waste": 0.10}
      ],
      "mo_personas": [
        {"catalog_code": "MO-OF", "workers": 4, "days_formula": "=QTY/3", "social_charge": 0.25}
      ],
      "mo_equipos": [...],
      "subcontratos": [...]
    }
  }]
}
```

### Etapa 3: Auto-matching de precios + cálculo cascada
- Al crear items con recursos, buscar cada catalog_code en los catálogos del usuario
- Asignar precios automáticamente
- Calcular en cascada: recursos → precio unitario → × cantidad → directo → indirectos → beneficio → impuestos → neto

### Etapa 4: Cascada de indirectos real
Replicar la estructura del Excel:
- Subtotal 01 (Directos)
- + Imprevistos % + Estructura % + Jefatura % = Subtotal 02
- + Beneficio % SOBRE Subtotal 02 = Subtotal 03
- + Ingresos Brutos % + Imp. Cheque % SOBRE Subtotal 03
- + IVA % = Total Final

### Etapa 5: Frontend — Vista de análisis unitario
- En ItemDetail, mostrar la composición del item (las 5 secciones)
- Permitir editar cantidades, agregar/quitar recursos
- Recálculo automático al editar

### Etapa 6: Items precalculados + histórico
- Importar composiciones históricas de los Excels existentes
- Biblioteca de items tipo: "Hormigón H21 columnas", "Mampostería LP18", etc.
- Al crear un presupuesto nuevo, poder elegir items de la biblioteca
